{% comment %} Collaboration Panel — the suite-wide per-record diligence surface. Composes five existing keel/components/ sub-includes in fixed order: 1. claim_row.html (who's the lead) 2. collaborator_list.html (LEAD / CONTRIBUTOR / REVIEWER / OBSERVER) 3. comment_section.html (internal-vs-external notes) 4. workflow_transitions.html (current state + available transitions) 5. attachment_list.html (uploads + Manifest-signed PDFs) The right-rail Activity + Quick Info are NOT inside this orchestrator — the calling detail page composes them separately so product-strength content above the panel and chrome around it stay product territory. This panel IS the per-record scaffolding underneath; see docs/design/collaboration-panel-2026-05.md for the spatial contract. Manifest, which has no collaboration semantics by design, does not include this panel at all — that's the right move; the panel is opt-in. Collapsed-by-default mode (Admiralty carve-out): Pass `collapsed=True` to wrap the whole panel in a
element with a single-line summary header. Designed for products where workflow + statutory clock + legal procedure dominate the main column; collaboration is one click away, not zero. See the Admiralty section of docs/design/collaboration-panel-2026-05.md for the rationale (statutory clock pressure, legal defensibility, requester-facing risk — NOT "buyer suspicion of collaboration-first tools", which Codex flagged as the wrong framing). Visibility / authorization (TWO requirements — both must hold): 1. Callers **MUST** pass querysets pre-filtered for the current user's visibility. The collapsed-mode summary header (`{{ collaborators|length }}`, `{{ notes|length }}`, `{{ attachments|length }}`) AND the count badges inside each sub-include reflect the INPUT queryset BEFORE row-level guards run. An unfiltered queryset shows "3 members · 2 notes · 1 file" to a non-staff user when internal-only rows make those counts wrong — the user learns staff-only content exists even though contents stay hidden. For external-applicant detail pages (Harbor's applicant portal, future Admiralty requester portal), pre-filter at the view layer. 2. The defensive per-row filters in `comment_section.html` and `attachment_list.html` are the SAFETY NET for rule 1 — they hide internal row CONTENTS at render time.
