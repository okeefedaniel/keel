{% comment %} Attachment list — renders an AbstractAttachment queryset with optional upload form. Distinguishes Manifest-signed PDFs (from the signing handoff roundtrip) and internal-vs-external visibility. Usage: {% include "keel/components/attachment_list.html" with attachments=record.attachments.all upload_action=upload_url csrf_token=csrf_token %} Each attachment must come from a model extending AbstractAttachment (fields: file, filename, content_type, size_bytes, description, visibility, source, uploaded_by, uploaded_at, manifest_packet_uuid). `upload_action` is optional — pass the URL the upload form should POST to, or omit to render read-only. Visibility contract (defense in depth): Callers SHOULD pass a queryset filtered by visibility for the current user (e.g. exclude visibility='internal' for non-staff). The per-row guard below is a SAFETY NET — if the caller forgets, the template still hides internal-flagged rows from non-staff at render time. Requires `django.template.context_processors.request` in TEMPLATES OPTIONS (suite default). If request is missing, `request.user.is_staff` resolves to False — the safe default. {% endcomment %} {% load humanize %}
