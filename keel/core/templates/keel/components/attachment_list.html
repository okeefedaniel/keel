{% comment %} Attachment list — renders an AbstractAttachment queryset with optional upload form. Distinguishes Manifest-signed PDFs (from the signing handoff roundtrip) and internal-vs-external visibility. Usage: {% include "keel/components/attachment_list.html" with attachments=record.attachments.all upload_action=upload_url csrf_token=csrf_token %} Each attachment must come from a model extending AbstractAttachment (fields: file, filename, content_type, size_bytes, description, visibility, source, uploaded_by, uploaded_at, manifest_packet_uuid). `upload_action` is optional — pass the URL the upload form should POST to, or omit to render read-only. Visibility contract (TWO requirements — both must hold): 1. Callers **MUST** pass a queryset filtered by visibility for the current user (e.g. `.exclude(visibility='internal')` for non-staff). The count badge `{{ attachments|length }}` reflects the INPUT queryset BEFORE the per-row guard runs — an unfiltered queryset shows "Attachments (5)" to a non-staff user when only 3 will render. That count discrepancy is a metadata leak: the user learns "there are 2 staff-only files I can't see" even though their content is hidden. For external-applicant detail pages this matters; pre-filter or override the badge in your view. 2. The per-row `{% if att.visibility != 'internal' or request.user.is_staff %}` guard below is the SAFETY NET for rule 1 — if the caller forgets to filter, internal-flagged row CONTENTS stay hidden from non-staff. It does NOT fix the count badge. Requires `django.template.context_processors.request` (suite default). If request is missing, `request.user.is_staff` resolves False — safe default (hide internal). {% endcomment %} {% load humanize %}
